PHOTONPAY API FACTS (from official docs):
- Token: POST /oauth2/token/accessToken with Authorization: basic Base64({appId}/{appSecret})
- Business calls use header: X-PD-TOKEN
- Signed write calls use header: X-PD-SIGN (MD5withRSA sign of request.body, then Base64 encode)
- Empty-body GET/query calls do NOT require body signing
- Subscription checkout: POST /commerce/openApi/v1/subscription/cashierSession
- One-time checkout: POST /txncore/openApi/v4/cashierSession
- Query transaction: GET /txncore/openApi/v4/s2sQuery
- Cashier JS UAT: https://cdn1.uat.photontech.cc/PhotonPay.js
- Cashier JS Production: https://cdn.photonpay.com/PhotonPay.js
- Cashier returns to redirectUrl with: status=[canceled/succeed/failed/pending]&reqId={reqId}
- Webhook uses X-PD-SIGN header + PhotonPay public key verification
- Webhook headers: X-PD-NOTIFICATION-CATAGORY, X-PD-NOTIFICATION-TYPE
- Webhook success response: { "roger": true }
- Payment notification types: sale, auth, capture, refund, void, mit
- Subscription notification types: subscription.created, subscription.updated, subscription.canceled, subscription.paused, subscription.resumed, subscription.trial_will_end
